## 摘要
基于大规模数据训练的视觉-语言-动作（VLA）模型取得了显著进展，但在部署时仍易受到分布偏移的影响。近期VLA模型表明，提示可作为调控策略行为的有效接口，但现有基于提示的调控通常依赖外部引导。这自然引发一个问题：能否通过优化提示实现VLA的测试时训练（TTT），使得调控接口本身能够从交互中学习并适应？我们通过TTT-VLA回答了这一问题，这是一种基于隐式提示优化（LPO）的测试时训练框架。在训练阶段，通过额外的代理任务学习隐式提示，为策略学习提供额外的学习条件信号。在测试时，通过从当前环境收集交互数据，并利用代理任务的自监督信号仅优化隐式提示（不修改策略本身）来执行TTT。在SimplerEnv上的实验表明，所提方法在单具身和多样具身设置中均能持续提升任务成功率。进一步分析显示，性能提升主要源于修正少量关键决策，而非全面改变策略行为。这些结果表明，LPO为基础操作策略的部署时改进提供了一条有效且实用的路径。

## Abstract
Vision-Language-Action (VLA) models trained on large-scale data have made remarkable progress, but they remain vulnerable to distribution shifts at deployment time. Recent VLA models suggest that prompts can serve as an efficient interface for steering policy behavior, but existing prompt-based steering typically relies on external guidance. This raises a natural question: can test-time training (TTT) for VLA be achieved by optimizing a prompt, so that the steering interface itself can be learned and adapted from interaction? We address this question with TTT-VLA, a test-time training framework based on Latent Prompt Optimization (LPO). During training, the latent prompt is learned with an additional proxy task, providing an extra learned conditioning signal for policy learning. At test time, TTT is performed by collecting interaction data from the current environment and optimizing only the latent prompt on those data using the proxy task's self-supervised signal, without modifying the policy itself. Experiments on SimplerEnv demonstrate that the proposed method consistently improves task success rates in both single- and multi-embodiment settings. Further analysis shows that the gains arise primarily from correcting a small number of critical decisions rather than globally altering policy behavior. These results suggest that LPO provides an effective and practical pathway for deployment-time improvement of foundation manipulation policies.

- **研究动机**：Vision-Language-Action (VLA) 模型在大规模数据预训练后表现出强大能力，但部署时面对环境分布偏移（如视觉条件、场景、机器人本体变化）依然脆弱。

- **核心问题**：能否通过优化一个可学习的隐式提示（latent prompt）来实现 VLA 的测试时训练（Test-Time Training, TTT），使提示接口本身能从部署交互中自适应？

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
- 将 VLA 策略表示为条件策略 \(\pi_\theta(a|o, c)\)，其中 \(c\) 包括显式条件（语言、状态、图像等）和一个可学习的隐式提示 \(z \in \mathbb{R}^{n \times d}\)（\(n\) 个 prompt token）。
- 通过一个代理任务（proxy task）——**状态接地（State Grounding）**——来驱动隐式提示的学习和测试时优化。
- 训练时联合优化策略主损失和代理损失；测试时仅用代理损失优化隐式提示，策略骨干冻结。

### 关键技术细节
- **代理任务（State Grounding）**：使用额外的“状态专家”（state expert）预测机器人末端执行器位置和夹爪状态。该任务采用条件流匹配（conditional flow matching）目标：
  \[
  \mathcal{L}_{sg}(\theta,z) = \mathbb{E}_{(o,c,s), s_0, t} \left[ \| v^\theta_s(s_t, t; o, c) - u^s_t \|_2^2 \right]
  \]
  其中 \(s_1 = s\) 为真实状态，\(s_0\) 为噪声，\(s_t = (1-t)s_1 + t s_0\)，\(u^s_t = s_0 - s_1\)。
- **网络结构**：基于 Mixture-of-Transformers (MoT) 架构，包含一个动作专家和一个状态专家。隐式提示通过硬路由器（hard router）分配给不同专家。
- **训练策略**：
  - 单具身设置：两阶段训练，第一阶段屏蔽隐式提示到动作专家的注意力（概率1），第二阶段以0.5概率随机丢弃（random drop），避免隐式提示过度耦合到动作预测。
  - 多样具身设置：不同本体对应不同的隐式提示，利用本体身份自然分区。
  - 梯度约束：状态接地的梯度仅流向状态专家和隐式提示；动作损失的梯度流向 VLM 主干和动作专家。
- **测试时训练流程**：
  - 在部署环境收集交互数据（包含成功和失败轨迹）存入缓冲区。
  - 冻结策略骨干（动作专家、状态专家、VLM 主干），仅用代理损失 \(\mathcal{L}_{sg}\) 优化隐式提示 \(z\)：
    \[
    z \leftarrow z - \eta \nabla_z \mathcal{L}_{sg}
    \]
  - 采用离线批量更新（batch size 128，学习率 1e-5），每任务优化固定步数（WidowX 500 步，Google Robot 1000 步）。

### 算法流程（文字说明）
1. **训练阶段**：
   - 从预训练的 π0.5 初始化模型。
   - 联合优化动作损失 + 状态接地损失，同时应用随机丢弃和梯度约束。
   - 学习得到隐式提示 \(z\)（单具身为一个，多样具身为每个本体一个）。
2. **测试时训练阶段**：
   - 在目标环境中运行冻结策略，收集交互轨迹（约 800～2000 条）。
   - 使用收集的数据，仅更新隐式提示（多个步数迭代）。
   - 更新后的提示用于后续评估。

## 6. 论文的主要结论与发现

- **主要结论**：隐式提示优化（LPO）能够作为 VLA 模型部署时改进的有效接口。TTT-VLA 在单具身和多样具身设置中均一致提升任务成功率。
  - 单具身 WidowX：π0.5 + SG-LP + TTT 达到 67.4%（对比基线 π0.5 的 51.1%）。
  - 单具身 Google Robot：visual matching 从 67.5% 提升到 72.4%；variant aggregation 从 58.1% 提升到 60.1%。
  - 多样具身：平均成功率从 22.8% 提升到 31.6%。
- **关键发现**：性能提升主要来源于**关键决策矫正（Critical Decision Steering）**，即 TTT 修正了少量关键的、阶段转换处的决策错误（如抓取后是否抬臂、放置时是否对齐），而非整体改变整个轨迹。
- **其他发现**：
  - 离线 TTT 比在线 TTT 更稳定，批量大小更重要。
  - 中等学习率 (1e-5) 效果最佳。
  - 随机丢弃训练对单具身设置至关重要，可避免隐式提示对策略的干扰。

- **创新性**：将测试时训练引入 VLA 领域，通过轻量级的隐式提示接口实现部署时自适应，避免修改主干参数。
- **实用性**：代理任务（状态接地）在训练和测试时均可获取（无需额外人工标注），自监督信号实用。

## 8. 不足与局限

- **无真实世界部署验证**：所有实验均在 SimplerEnv 仿真中进行，未在真实机器人上测试 TTT 流程（交互收集、优化、再评估），实际部署中的稳定性、安全性等问题未涉及。
- **代理任务单一**：仅使用状态接地任务，未探索其他自监督代理（如未来状态预测、逆动力学、表示预测等），可能限制了性能上限。
- **未直接对比先前 TTT 方法**：与 WorldAgen 等并发工作由于代码不可用或设置差异未能进行公平比较。
- **在线 TTT 不稳定**：全在线更新（batch size 1）容易导致提示坍塌，目前仅使用离线 TTT，限制了实时自适应能力。
- **部署代价非零**：虽然只有少量参数更新，但仍需收集数百条轨迹并执行几百步优化，交互和计算成本尚存。
- **增益模式局限**：改进主要体现为局部关键决策修正，而非长时复杂行为重建，可能限制在更复杂任务上的提升空间。
- **梯度约束与随机丢弃增加了训练复杂度**：需要精细调节，可能在更广泛模型上迁移性待验证。
