## 摘要
视觉-语言-动作（VLA）模型为通用机器人策略提供了一种有前景的范式，但其适应性受到数据效率低下和泛化能力差的阻碍。我们认为这些瓶颈源于主流的直接指令到控制映射，该映射迫使模型记忆整体轨迹而非可复用的运动模式（即基元）。我们提出PrimitiveVLA，一个将范式转向以基元为中心的“分解与组装”范式的框架。在共享的多模态规范表征（MCR）支持下，PrimitiveVLA统一了两个阶段：（1）微调阶段分解，使用自动化流水线将演示分解为可复用的基元；（2）推理阶段组装，采用基于VLM的规划器和LLM生成的开关模块实现鲁棒的闭环执行。通过将任务分解为可复用的基元，PrimitiveVLA使VLA模型能够学习不变的运动模式而非任务特定的轨迹。大量实验表明，我们的框架提高了数据效率，并在未见任务和长时域任务上实现了卓越的零样本泛化。

## Abstract
Vision-Language-Action (VLA) models offer a promising paradigm for generalist robotic policies, yet their adaptation is hindered by data inefficiency and poor generalization. We argue that these bottlenecks stem from the prevailing Direct Instruction-to-Control Mapping, which forces models to memorize monolithic trajectories rather than reusable motion patterns, i.e., primitives. We propose PrimitiveVLA, a framework that shifts this paradigm toward a Primitive-Centric Disassemble & Assemble paradigm. Supported by a shared Multimodal Canonical Representation (MCR), PrimitiveVLA unifies two phases: (1) Fine-tuning-phase Disassembly, which uses an automated pipeline to disassemble demonstrations into reusable primitives; and (2) Inference-phase Assembly, which employs a VLM-based planner and an LLM-generated switch module for robust closed-loop execution. By disassembling tasks into reusable primitives, PrimitiveVLA enables VLA models to learn invariant motion patterns instead of task-specific trajectories. Extensive experiments show that our framework improves data efficiency and achieves superior zero-shot generalization across unseen and long-horizon tasks.

# 论文总结：PrimitiveVLA: Learning Reusable Motion Primitives for Efficient and Generalizable Robotic Manipulation

## 1. 核心问题与整体含义（研究动机和背景）

- **背景**：视觉-语言-动作（VLA）模型（如 RT-2、OpenVLA）通过互联网规模数据预训练，具备开放世界操作的潜力，但在微调下游领域时面临**数据效率低**和**跨任务泛化差**两大瓶颈。
- **核心问题**：现有主流范式是**直接指令到控制映射（Direct Instruction-to-Control Mapping）**，它将高层的任务指令直接映射到低层的控制动作，导致模型被迫记忆整体轨迹而非可复用的运动模式（即“基元”，primitive）。这造成两个后果：
  - 数据复用困难：例如，“打开橱柜”与“打开微波炉”的知识无法迁移，每个新任务都需要昂贵的演示。
  - 泛化脆弱：模型倾向于记忆场景特定的视觉模式，而非鲁棒的物理运动模式，导致在未见任务上表现显著下降（如 AGNOSTOS 基准所示）。

## 2. 方法论

### 核心思想：基元中心分解与组装（Primitive-Centric Disassemble & Assemble Paradigm）

- 将任务轨迹分解为一组有限的、可复用的基元（如 Grasp、Pull、Twist 等），微调时让 VLA 学习这些基元，推理时利用规划器组装基元解决新任务。
- 通过共享的**多模态规范表征（Multimodal Canonical Representation, MCR）**统一微调和推理阶段。

### 关键技术细节

1. **预定义基元库（11个基元）**：
   - 空间传输类：Grasp、Place、Lift、Move
   - 接触与交互类：Push、Pull、Insert、Press
   - 方向类：Twist、Tilt、Rotate

2. **微调阶段分解（Fine-tuning Disassembly）**：
   - **基元序列推理（Primitive Sequence Reasoning）**：利用 VLM（Qwen3-VL）根据任务指令、采样视频帧和基元库，推理基元执行顺序，得到（指令，序列）对并索引到分解库 D 中。
   - **状态边界分割（State-Based Boundary Segmentation）**：利用 LLM（DeepSeek-V3）根据基元的物理定义和数据集相关的基元集合，生成 Python 代码来定义每个基元的终止条件（例如，Grasp 终止于夹爪宽度恒定且末端高度开始上升）。代码通过局部窗口的状态统计实现鲁棒边界定位。
   - **学习时应用 MCR**：将任务特定指令映射为规范基元指令（如“grasp the object with green mask”），观测图像应用物体中心掩码（通过 SAM + Cutie 跟踪），确保不同场景下相同基元的输入格式统一。

3. **推理阶段组装（Inference Assembly）**：
   - **基元规划器（Primitive Planner）**：从分解库 D 中检索与测试任务最相似的 3 个（指令，序列）对，连同测试指令、初始观测和基元定义输入 VLM，通过上下文学习生成合理基元序列。
   - **基元开关（Primitive Switch）**：LLM 根据基元定义和之前的分割代码，生成轻量级 Python 开关函数 `f_switch`，监控滑动历史窗口（10步）的统计趋势，实现稳定的基元间切换。执行时采用双线程架构：执行线程输出动作，监控线程评估开关条件。

4. **多模态规范表征 MCR**：
   - **语义统一**：将同一基元类型的所有样本映射到唯一规范指令（如 “grasp the object with green mask”）。
   - **视觉兼容**：使用物体中心掩码（SAM + Cutie 跟踪）获得 `õ_i = o_i ⊗ M_i`，任务特定上下文由掩码区域携带，背景纹理被抑制。

5. **训练目标**：标准 VLA 的损失函数 `L_prim` 在分解后的基元数据上优化，使模型学习不变的运动模式。

1. **性能提升**：PrimitiveVLA 在 Libero-90 上将 OpenVLA 性能提升 9.2%（从 70.6% 到 79.8%），OpenVLA-OFT 提升 5.0%（从 89.7% 到 94.7%），π0.5 保持竞争力。
2. **数据效率**：使用 50% 数据时，PrimitiveVLA 可达到或超越基线 100% 数据的性能（如 OpenVLA + Ours 50% 数据 80.3% vs 基线 70.6%）。
3. **零样本泛化**：
   - 在未见任务（Libero-90-Novel）上，OpenVLA 从 7.38% 提升至 45.5%（6×），OpenVLA-OFT 从 13.5% 提升至 71.0%，π0.5 从 56.0% 提升至 75.5%。

## 7. 优点

- **范式创新**：从“直接指令-控制映射”转向“基元分解-组装”，从根本上解决了 VLA 数据复用和泛化瓶颈。
- **自动化流水线**：无需人工标注，利用 VLM+LLM 自动完成基元序列推理、边界分割和开关代码生成，实用性强。
- **模型无关性**：在多个骨干模型（OpenVLA、OpenVLA-OFT、π0.5）上均有效，并兼容不同架构（Transformer、Flow Matching）。
- **高效的 MCR 设计**：通过规范指令和物体掩码，在保持任务上下文的同时消除了场景纹理干扰，使相同基元在不同任务中保持一致表示。
- **推理轻量**：基元开关代码执行极快（通常 <10ms），掩码跟踪使用轻量级 Cutie（9~30ms），整体额外延迟可忽略，不影响实时控制。
- **消融实验设计合理**：清晰分离了两个核心组件（分解与 MCR），并给出了不同任务类型下的贡献差异，有助于理解机制。

## 8. 不足与局限

- **基元库人为预定义**：当前基元库（11 个基元）基于运动学分类，覆盖了标准夹爪操作，但未覆盖高度专业化的灵巧操作（如手指精细抓取、拧动等）。未来需探索无监督基元发现以动态扩展动作空间。
- **对 VLM/LLM 噪声的鲁棒性**：论文提到分解过程偶发噪声（<5% 在 LIBERO, <10% 在 RLBench），虽被后续规则缓解，但高频噪声仍可能影响性能。未充分量化噪声对最终成功率的影响。
- **真实世界范围有限**：真实世界仅测试 11 个任务，且场景较为静态（桌子、抽屉、橱柜），未涉及移动物体、光照变化或人类动态干扰。评估规模偏小。
- **依赖物体掩码质量**：MCR 依赖 SAM+Cutie 的掩码跟踪质量，在遮挡严重、物体外观变化大或背景杂乱时可能退化（论文未深入讨论失败案例）。
- **缺乏与其他层次化方法的直接对比**：实验对比了多种 VLA 基线，但未与同样强调分层/基元的方法（如 Pivot-R、StARe VLA、π0.7）进行直接定量比较（仅在相关工作中提及）。
- **泛化仅限于零样本**：未探索少样本适应或持续学习场景，也未测试迁移到完全不同的机器人本体。
- **计算资源需求**：虽然推理轻量，但微调阶段仍需 8 张高端 GPU（L40S/A100）训练数十小时，资源门槛不低。
