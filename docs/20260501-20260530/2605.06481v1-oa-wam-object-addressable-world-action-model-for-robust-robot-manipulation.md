## 摘要
世界动作模型通过联合预测场景演变和机器人动作来增强视觉-语言-动作策略，但现有方法通常将预测的世界表示为整体图像、视频token或全局潜在向量。当指令指向特定对象时，这些表示难以被动作解码器寻址，尤其是在对象身份与上下文纠缠的场景偏移下。我们提出OA-WAM，一种面向鲁棒机器人操作的可寻址对象世界动作模型。OA-WAM将每一帧分解为N+1个槽状态，包括一个机器人槽和N个对象槽。每个槽包含一个持久地址向量和一个时变内容向量，并与文本、图像、本体感知和过去动作token在块因果序列中融合。世界头预测下一帧的槽状态，而流匹配动作头在同一前向过程中解码16步连续动作块。通过仅使用地址键进行跨槽注意力路由，并在每个Transformer层重置地址切片，实现了可寻址性，将作用于哪个对象与该对象当前是什么分离，而无需添加额外token。OA-WAM在LIBERO（97.8%）和SimplerEnv（79.3%）上匹配了强大的VLA和WAM基线，在最具相关性的LIBERO-Plus几何轴上达到最先进性能，并在七轴聚合上保持竞争力。因果槽干预测试产生0.87的交换绑定余弦，而整体基线最多为0.09。这些结果表明，可寻址对象状态为场景扰动下的鲁棒世界-动作建模提供了有效接口。

## Abstract
World Action Models (WAMs) enhance Vision-Language-Action policies by jointly predicting scene evolution and robot actions, but existing methods usually represent the predicted world as holistic images, video tokens, or global latents. These representations are difficult for an action decoder to address when an instruction refers to a particular object, especially under scene shifts where object identity is entangled with context. We propose OA-WAM, an Object-Addressable World Action Model for robust robot manipulation. OA-WAM decomposes each frame into N+1 slot states, with one robot slot and N object slots. Each slot contains a persistent address vector and a time-varying content vector, and is fused with text, image, proprioception, and past-action tokens in a block-causal sequence. A world head predicts next-frame slot states, while a flow-matching action head decodes a 16-step continuous action chunk in the same forward pass. Addressability is enforced by routing cross-slot attention through address-only keys and resetting the address slice at every transformer layer, separating which object to act on from what that object currently is without adding extra tokens. OA-WAM matches strong VLA and WAM baselines on LIBERO (97.8%) and SimplerEnv (79.3%), reaches state-of-the-art performance on the most relevant LIBERO-Plus geometric axes, and remains competitive on the seven-axis aggregate. A causal slot-intervention test yields a swap-binding cosine of 0.87, versus at most 0.09 for holistic baselines. These results suggest that addressable object states provide an effective interface for robust world-action modeling under scene perturbations.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：现有世界动作模型（World Action Models, WAM）通过联合预测场景演变和机器人动作来增强视觉-语言-动作（VLA）策略，但它们通常将预测的世界表示为整体图像、视频 token 流或全局潜在向量。当指令指向特定对象时（如“把红色杯子放在绿色托盘上”），这种整体表示使得动作解码器难以准确定位目标对象，尤其是在对象身份与周围上下文纠缠的场景偏移下（如光照、背景、布局、相机视角变化等），性能急剧下降。
- **核心缺失**：论文将其归结为**缺乏对象可寻址性（object addressability）**——世界状态不仅应预测未来，更应为语言条件动作生成提供稳定、可查询的每对象状态，使“作用于哪个对象”与“该对象当前是什么”在架构上解耦。
- **意义**：提出一种将世界模型显式组织为可寻址对象状态的方法，为场景扰动下鲁棒的操作提供了一条有效路径。

## 2. 方法：核心思想、关键技术细节

### 核心思想
将每一帧分解为 **N+1 个槽（slots）**（1 个机器人槽 + N 个对象槽），每个槽由两部分组成：
- **地址向量 (addr)**：只包含物体身份信息，在一次片段内固定，由语言标签和初始 DINOv3 特征计算。
- **内容向量 (content)**：包含时变的状态（视觉外观、位姿等），每帧更新。

在 Transformer 各层中，**跨槽注意力仅通过地址子向量路由**（addr-only key projection），加上**每层残差流中地址切片重置**，实现“对象寻址”与“对象内容”的解耦。

### 关键模块
1. **六路 token 化**：
   - 文本（BPE token）
   - 图像（Chameleon VQ-GAN 编码）
   - 对象-槽（SAM3 + DINOv3 分割 + 位姿，通过槽适配器投影）
   - 本体感知（离散化 7 维）
   - 过去动作（离散化）
   - 词性短语（仅用于 SAM3 提示，不入主干）
2. **槽向量构造**：
   - \( s_t^k = [\text{addr}_k \in \mathbb{R}^{32}; \text{cnt}_t^k \in \mathbb{R}^{256}; \pi_t \in \mathbb{R}^{16}; \rho_k \in \mathbb{R}^{16}] \)（总共 320 维）
   - 地址 addr_k 一次计算、冻结；内容 cnt_t^k 每帧更新；π_t 是帧时间嵌入；ρ_k 是角色分类。
3. **槽感知主干**（7B Chameleon 变体）：
   - 32 层 Transformer，隐藏维度 4096，32 个注意力头。
   - **地址约束**：在每个 Transformer 层，对 **槽位置的 key 投影**只取输入向量的前 32 维（地址部分）；非槽位置保持标准全维 key。残差流中每层后置钩子将前 32 维覆盖为原始缓存地址。
   - 同一帧内的槽间注意力双向（置换等变），帧间块因果。
4. **预测头**：
   - **世界头**：通过两个 MLP 预测下一帧的每个非机器人槽的内容（256 维 MSE）和位姿（9 维 MSE）。
   - **动作头**：流匹配 MLP（8 块残差网络），以序列末尾的 [ACT_Q] 隐藏状态为条件，预测 16 步连续动作 chunk；推理时用 4 步前向欧拉积分解码。
   - **辅助图像 VQ 头**：复用主干 lm_head 预测下一帧 VQ token，引入加权交叉熵。
5. **训练目标**：
   - 总损失 \( L = L_{\text{act}} + \lambda_w L_{\text{world}} + \lambda_v L_{\text{vq}} + \lambda_c L_{\text{compose}} + \lambda_r L_{\text{role}} \)
   - \(L_{\text{world}}\)：每槽下一帧 MSE；\(L_{\text{compose}}\)：对干扰物排列/插入的不变性正则；\(L_{\text{role}}\)：对目标/参考槽的弱监督对齐（仅前 50k 步）。
   - 权重 \(\{\lambda_w=0.5, \lambda_v=0.04, \lambda_c=0.1, \lambda_r=0.05\}\)。\(\lambda_c\) 在前 30% 训练中线性升温，\(\lambda_r\) 在 50k 步后设为 0。

### 伪代码简要流程（算法 1 和 2）
- 推理：每步通过 SAM3/DINOv3 获取槽，构建序列，经槽感知主干得到 [ACT_Q] 隐藏状态，再经世界头和动作头输出。
- 训练：第二阶段（Stage II）微调，使用 LIBERO 标准演示，冻结主干并加载 LoRA（rank=32），更新槽适配器、世界头、动作头。

## 6. 主要结论与发现

1. **可寻址对象状态关键**：在几何置换的鲁棒性轴上（Camera、Robot init、Layout），OA-WAM 达到新 SOTA（Geo Avg 84.3%，比 π0.5 高 4.8%），表明地址路由机制有效防止场景上下文干扰目标选择。
2. **分布内性能不降**：LIBERO 上达 97.8%，SimplerEnv 上达 79.3%，证明对象分解本身未牺牲标准测试性能。
3. **因果绑定验证**：交换目标地址后，OA-WAM 的末端执行器轨迹余弦对齐为 0.87，而所有整体基线 ≤ 0.09；关掉地址约束后降至 0.19，说明绑定确实来自地址子空间路由。
4. **感知瓶颈**：在 Sensor Noise 轴上落后 17.1%，因 SAM3 在光度失真下提取失败，而非政策层问题。
5. **OA约束是特定于 OOD 的归纳偏置**：去掉地址约束仅影响几何鲁棒性（LP camera下降 20%），而 LIBERO 标准仅降 1.5%，说明不是通用的容量增益。

## 7. 优点

- **架构创新**：提出地址/内容分离的槽表示和参数无关的注意力约束，无需额外参数即可实现对象寻址。
- **实验设计严谨**：包含丰富的消融、因果验证、归因分析，能够明确量化各个组件贡献。
- **代码与检查点公开**（论文提及）：使实验可复现。
- **可解释性强**：通过干预实验、注意力可视化等展示了“地址子空间”的实际作用机制。
- **计算资源公开透明**：给出详细三阶段训练配置和总耗。

## 8. 不足与局限

- **仿真环境限制**：所有实验均在模拟器中进行，真实机器人部署的鲁棒性尚未验证。
- **感知瓶颈**：冻结的 SAM3 + DINOv3 在小、反射、透明、遮挡或运动模糊物体上易失败，是 Sensor Noise 轴差的主因（29.7% 失败源于感知）。感知延迟（138 ms）也限制了实时控制率。
- **假设局限**：干扰一致性损失假设目标/干扰物弱耦合，若物理上干扰物阻挡目标则失败。
- **地址空间容量**：32 维地址在 LIBERO 场景下利用率 ≤ 3%，但更大场景或类别数可能是瓶颈。
- **未探索**：无真实世界实验；动作输出仅支持 WidowX 机械臂构型；对比方法中缺少基于槽的 WAM（如 OCWM），但论文说明现有基于槽的方法仍将身份与内容混合，OA-WAM 是第一个在架构上解耦的。
