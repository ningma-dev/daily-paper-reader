## 摘要
视觉-语言-行动策略将语言和视觉输入转化为机器人动作，其隐藏表示直接影响闭环行为。然而，来自语言和视觉-语言模型的机制可解释性工具无法直接迁移到视觉-语言-行动策略：输出是机器人动作而非人类可读的词元，干预只能通过昂贵的闭环 rollout 测试。我们提出了一种基于事件锚定的可解释性流水线，将稀疏自编码器特征分析锚定到行为事件而非文本上下文。利用视觉、状态和时间线索，在每个任务内对末端执行器关键帧进行聚类，将稀疏自编码器特征与行为显著事件关联，并通过可选的视觉-语言模型注释与语义上下文关联。据我们所知，我们的流水线是首批将基于稀疏自编码器的视觉-语言-行动策略分析锚定到闭环行为事件的工作之一。在两种仿真架构和一项真实机器人研究中，基于事件锚定的排序在 OpenVLA 上产生了最强的因果效应，并迁移到了 π₀.₅ 的连续动作块中。稀疏自编码器是一种稀疏但不完美的干预基础：其可用性随架构和干预位置而变化，激进的干预揭示了安全性和可解释性限制。总体而言，基于事件锚定的稀疏自编码器分析成为行为锚定的视觉-语言-行动策略可解释性实用起点，为后续工作提供了方向：超越动作对齐坐标的稀疏自编码器特征、更细粒度的闭环评估，以及高风险视觉-语言-行动策略部署的安全干预。代码见 \url{https://github.com/xc-j/Event-SAE}。

## Abstract
Vision-Language-Action (VLA) policies translate language and visual inputs into robot actions, where their hidden representations directly shape closed-loop behavior. However, mechanistic interpretability tools from language and vision-language models do not transfer cleanly to VLAs: outputs are robot actions rather than human-readable tokens, and interventions can only be tested via expensive closed-loop rollouts. We propose an event-grounded interpretability pipeline that anchors SAE feature analysis to behavioral events rather than text contexts. End-effector keyframes are clustered within each task using visual, state, and temporal cues, linking SAE features to behaviorally salient events and, via optional VLM annotations, to semantic context. To our knowledge, our pipeline is among the first to ground SAE-based VLA analysis in closed-loop behavioral events. Across two simulation architectures and a real-robot study, event-grounded ranking yields the strongest causal effects on OpenVLA and transfers to the continuous action chunks of $π_{0.5}$. SAE is a sparse but imperfect intervention basis: usability varies with architecture and intervention site, and aggressive intervention reveals safety and interpretability limits. Overall, event-grounded SAE analysis emerges as a practical starting point for behavior-anchored VLA interpretability, motivating future work on SAE features beyond action-aligned coordinates, finer-grained closed-loop evaluation, and safe interventions for high-stakes VLA deployments. Code is available at \url{https://github.com/xc-j/Event-SAE}.

## 1. 核心问题与整体含义（研究动机和背景）

视觉-语言-动作策略（VLA）将语言指令和视觉观测映射为机器人动作，其隐藏表示直接驱动闭环行为。然而，现有的从语言模型和视觉-语言模型（LLM/VLM）借鉴的机械可解释性工具无法直接迁移到VLA上，原因有二：

- **特征命名困难**：VLA的输出是连续或离散的动作（如笛卡尔坐标、关节位置），而非人类可读的词（token）。logit lens等方法无法直接投影到语义词汇空间。
- **行为验证昂贵**：LLM中一次干预就能从输出token读出语义变化，但VLA需要完整闭环rollout（多步动作）后才能评估结果，单步干预难以解读。

现有工作（如Swann等2026）虽然用稀疏自编码器（SAE）分析VLA，但特征标注依赖人工对齐激活痕迹与视频，不具扩展性。本文提出一种**事件锚定的SAE可解释性流水线**，自动从闭环rollout中提取运动学关键帧、聚合成行为事件，并利用这些外部事件（而非SAE内部统计）对每个SAE特征进行评分和因果验证，从而避免人工标注，实现全SAE基的自动化分析。

## 2. 方法论

### 核心思想
通过四个阶段将闭环VLA rollouts转化为基于事件锚定的SAE因果特征分析：

1. **SAE训练**：在残差流上训练BatchTopK稀疏自编码器，将密集隐藏状态分解为高维稀疏特征基。
2. **运动学关键帧提取**：使用Automatic Waypoint Extraction (AWE) 从末端执行器位置轨迹中提取关键时间步（keyframes），每个keyframe附带一个观察束（多帧图像、视频片段、任务元数据）。
3. **事件聚类**：将keyframe描述符（视觉嵌入+机器人状态+时间进展）进行任务局部的凝聚聚类，保留覆盖半数以上episode的簇，作为“事件”。可选地，利用VLM为每个簇标注短语和阶段标签。
4. **因果干预验证**：通过闭锁干预（closed-loop intervention）测试特征重要性。

### 特征排名策略
比较四种排名：

- **事件对齐排名**：衡量特征在事件时间窗内的激活形状是否匹配三种模板（脉冲、阶跃上升、阶跃下降），得分最高。
- **窗口均值排名**：事件时间窗内平均激活值。
- **任务均值排名**：整个任务所有时间步的平均激活值。
- **随机存活排名**：从存活特征中随机选取（排除其他排名所选特征）。

### 干预模式
- **重建挂钩**：用SAE编解码替换隐藏状态，评估保真度。
- **残差保持潜变量编辑**：修改目标特征（缩放因子α），只改动SAE可解释部分，保留重建误差。支持从抑制（α=0）到放大（α>1）。

### 关键技术公式
- SAE编码：`z = BatchTopK(W_enc h + b_enc)`，解码器重建`ŷ = W_dec z + b_dec`。
- 损失函数：`L_SAE = (1/B) Σ ||h - ŷ||² + λ_aux L_aux`。
- 干预编辑：`x' = x + Dec(z') - Dec(z) = Dec(z') + err(x)`。

## 3. 实验设计

### 策略和任务
- **策略**：OpenVLA（离散动作token）和 π0.5（连续动作块，其中PaliGemma视觉-语言骨干 + 动作专家AE）。
- **数据集/场景**：四个LIBERO套件（Spatial, Object, Goal, 10），共40个任务，每个任务50次rollout。
- **真实机器人**：使用Mobile ALOHA平台执行“接近红色/黄色薯片”任务，π0.5 LoRA微调。

### 对比方法

- **闭环保真度**：Hooked SR（重建成功率）。

## 6. 主要结论与发现

1. **SAE保真度依赖于架构和层**：OpenVLA早期层重建保真度好但闭环保真度低（Hooked SR=0%），仅最后一层（L31）保留部分成功率（34.8%）；π0.5的PG和AE几乎所有层保持≥95% Hooked SR。
2. **事件对齐排名最有效**：在OpenVLA L31，事件对齐排名选定特征使成功率下降21.2个百分点，显著优于随机存活（-1.3pp）；窗口均值和任务均值居中。
3. **π0.5的PG骨干对单特征编辑不敏感**：任何排名单特征零值干预后SR几乎无变化（降幅<3pp）；需联合丢弃16个特征才看到明显效果，且仅限早期层（L0）。
4. **π0.5的动作专家（AE）极为敏感**：几乎所有前3特征（包括随机存活部分）导致SR接近零，表明AE特征具有广泛的因果敏感性而非事件特异性。
5. **特征重叠分析**：事件对齐与窗口均值/任务均值选择的特征高度不重叠，说明时序形状（而非平均激活）提供额外信息。
6. **真实机器人实验中**：SAE抑制可调节颜色条件接近行为，效果依赖于层和强度，但强编辑会带来安全风险。

## 7. 优点

- **自动化程度高**：整个流程无需人工标注，从关键帧提取、聚类到特征评分自动完成，可扩展到大量特征。
- **事件锚定新颖且独立**：利用外部行为信号（运动学关键帧）作为锚点，避免SAE内部统计偏差，为VLA可解释性提供更直接的因果基础。
- **多架构验证**：测试了两种完全不同的VLA架构（离散token vs. 连续流匹配），揭示了可迁移性与架构依赖性，增加结论稳健性。
- **软干预剂量响应**：通过连续α扫描展示了SR随干预强度的单调变化，为安全校准提供可能。
- **真实机器人验证**：在真实物理平台上测试了SAE驱动的行为改变，并讨论了安全考量，增加了实践相关性。

## 8. 不足与局限

- **任务共享特征而非任务特定**：目标/非目标探针显示事件簇选取的特征在相同套件内对其它任务也有类似干扰效果，说明它们编码的是共享操作原语而非细粒度任务变量。
- **VLM语义利用不足**：VLM标注的阶段标签仅作为可视化辅助，并未用于排名或干预评分，语义结构被低估。
- **粗粒度的二元评估**：仅用成功率（SR）衡量干预效果，无法捕捉选择性行为改变（如抓取偏移但完成），也未记录轨迹层面差异。强编辑还会导致策略分布外漂移（off-distribution）。
- **安全限制**：真实机器人实验中只使用保守抑制以避免危险动作，限制了探索编辑空间的完整范围。
- **SAE活特征比例差异大**：OpenVLA深层只有3-8%存活，导致候选特征极少，可能遗漏重要但稀疏的特征。
- **干预基础不完美**：SAE特征本身可能多语义（polysemantic），且零值编辑可能导致行为崩溃而非精细控制，验证了“稀疏但非完美”的结论。
- **未对比其他可解释性方法**：如注意力分析、概念瓶颈模型等，难以定位SAE方法的相对优势。
