## 摘要
我们引入了COMET（用于高效树搜索的因果对象中心模型），这是一种基于模型的强化学习算法，可以在插槽结构的潜空间中进行蒙特卡洛树搜索。COMET将一个冻结的无监督对象中心编码器与基于Transformer的世界模型配对，其中通过一种新颖的动作-插槽融合机制将动作绑定到对象上，该机制用于插槽转换预测。策略和价值头使用对象因果注意力，通过学习到的每插槽相关性得分来调节令牌交互，从而使决策集中在任务相关实体上。COMET为MuZero风格的潜在规划增加了显式的对象级归纳偏差。在来自Object-Centric Visual RL基准、ManiSkill、Robosuite和VizDoom的八个视觉和动态多样化的任务中，COMET在训练早期阶段相比对象中心和整体基线获得了更高的平均标准化分数。

## Abstract
We introduce COMET (Causal Object-centric Model for Efficient Tree search), a model-based reinforcement learning algorithm that performs Monte Carlo Tree Search in a slot-structured latent space. COMET pairs a frozen unsupervised object-centric encoder with a transformer-based world model, in which actions are bound to objects through a novel action-slot fusion mechanism that is used in slot transition prediction. Policy and value heads use object-causal attention, modulating token interactions by learned per-slot relevance scores so that decision-making concentrates on task-relevant entities. COMET adds an explicit object-level inductive bias to MuZero-style latent planning. Across eight visually and dynamically diverse tasks from the Object-Centric Visual RL benchmark, ManiSkill, Robosuite, and VizDoom, COMET achieves a higher mean normalized score during the early stages of training compared to object-centric and monolithic baselines.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：在基于模型的强化学习（MBRL）中，学习准确的世界模型在视觉高维、非平稳、包含多交互对象的环境中非常困难。现有方法大多使用卷积编码器提取单一整体表示，无法捕获对象级结构和交互，导致小目标、动态背景或大量无关实体严重降低性能。人类通过将世界感知为离散对象来实现高效推理和规划，因此引入对象中心表示是克服这些困难的关键。
- **背景**：已有对象中心MBRL方法（如COBRA、FOCUS、OC-STORM）或依赖监督分割、或未显式建模对象交互和重要性。同时，在任意时刻只有少数对象与当前决策相关，区分任务相关对象与无关对象可以极大提升规划效率。由此，本文提出COMET：为MuZero风格的潜在规划添加显式对象级归纳偏置，通过因果注意力聚焦关键实体。

## 2. 方法论：核心思想、关键技术、公式等

- **核心思想**：COMET在插槽结构化潜空间中进行MCTS规划。它结合：
    1. 冻结的无监督对象中心编码器（SLATE / DINOSAUR / Slot Contrast），将图像观测映射为固定数量的对象插槽（包括背景）。
    2. 基于Transformer的世界模型，处理插槽序列并预测下一插槽和奖励。
    3. 策略和价值头使用对象因果注意力（Object-Causal Attention），通过学习的每插槽相关性得分（α）调节注意力，使决策集中在任务相关实体上。

- **关键技术细节**：
    - **动作-插槽融合（Action-Slot Binding）**：为解决从单一动作嵌入生成多个插槽的瓶颈，将动作嵌入与每个插槽独立拼接，然后通过共享MLP投影得到插槽条件动作嵌入，输入Transformer预测下一插槽。这实现了软性的动作与对象绑定。
    - **Transformer注意力掩码**：每个插槽可以关注同一时间步内的所有插槽、之前时间步的全部插槽以及对应位置的动作嵌入；动作嵌入关注自身及所有插槽。
    - **对象因果注意力**：基于预定义的因果图（目标令牌、因果对象、非因果对象），通过学习的因果得分α（0~1）调节缩放点积注意力，使得相关对象对目标令牌的注意力增强，无关对象被抑制。公式：CA = Norm( softmax( QK^T/√d ) ⊙ WGW^T ) V。
    - **训练目标**：MuZero风格，在潜空间展开K步，联合优化插槽预测（MSE）、奖励、策略和价值预测（交叉熵）。损失函数包含多个平衡系数（β_s, β_r, β_p, β_v）。
    - **MCTS规划**：在潜空间进行，使用学习到的预测函数提供先验，通过仿真次数生成改进策略目标。

- **算法流程**：观测通过冻结编码器提取插槽 → Transformer处理得到潜在表示 → 策略/价值头使用因果注意力输出动作分布和值 → 动作-插槽融合后通过Transformer预测下一插槽和奖励 → MCTS在潜空间搜索 → 更新模型参数。

## 3. 实验设计

- **数据集/场景**：8个视觉和动态多样化的任务，来自：
    - Object-Centric Visual RL benchmark：Object Goal, Object Interaction, Object Comparison, Property Comparison, Object Reaching（离散或连续动作，稀疏奖励，涉及目标对象与干扰物）。
    - ManiSkill：Cube Pushing（推方块到目标区域，密集奖励）。
    - Robosuite：Block Lifting（抓取并提升方块，密集奖励）。
    - VizDoom：Defend the Line（射击怪物，+1杀敌，-1死亡）。

- **Benchmark**：每个任务有特定的时间步预算（如500k或1M），评估指标为成功率或累积奖励。使用**平均归一化得分（Mean Normalized Score）**在多个环境上聚合性能，以消除尺度差异。

- **对比方法**：
    - 对象中心基线：OCRL（模型自由PPO）、OCDreamer、SOLD（仅连续控制，使用SAVi编码器）。

## 6. 主要结论与发现

- COMET在早期训练阶段（有限交互预算下）实现了平均归一化得分的提升，显著优于对象中心基线（OCRL、OCDreamer、SOLD）和整体基线（UniZero）。
- 在视觉简单且具有明确目标对象干扰的任务（如Object Goal, Object Comparison, Property Comparison）中，COMET表现最强，因为因果注意力能准确识别相关对象。
- 在动态复杂或对象交互深度较大的任务（如Defend the Line, Object Interaction）中，COMET与基线持平，归因于多数对象相关或因果链过长。
- 在视觉复杂且控制困难的任务（如Block Lifting, Cube Pushing）中，COMET表现中等，主要受限于对象中心编码器未能完美分割对象（如方块与背景合并）。
- 动作-插槽融合机制有效缓解了单动作嵌入成为瓶颈的问题，是实现对象中心世界模型的关键设计。

## 7. 优点

- **方法创新**：首次将对象因果注意力与MuZero风格MCTS结合，引入显式对象级归纳偏置；动作-插槽融合实现了软性的任务相关对象绑定，通用性强。
- **实验设计**：使用了多任务、多基线、多种子评估，并引入平均归一化得分处理尺度差异，使比较更有意义。提供了轨迹rollout和因果得分可视化，增强了可解释性。
- **资源效率**：冻结对象编码器减少了训练开销，在单GPU上即可完成实验，具有良好的实用性。
- **结果稳健**：在大部分任务上展示了样本效率优势，且与部分强基线（如OCDreamer）性能相当或更优。

## 8. 不足与局限

- **实验覆盖不足**：

- **偏差风险**：
    - 平均归一化得分可能夸大相对于基线的优势（归一化参数由最优方法决定）。
