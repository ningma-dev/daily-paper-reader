## 摘要
视觉-语言-动作（VLA）模型在机器人操作中展现出巨大潜力，但在边缘硬件上的实时部署仍然具有挑战性。在这项工作中，我们识别出VLM的视觉和上下文令牌是部署延迟的主要来源：对于以GEMM为主的投影算子，当模型维度固定时，计算量随输入令牌数量线性增长。基于这一观察，我们提出了RhinoVLA，一种面向部署的VLA模型，与辉羲R1边缘SoC协同设计。RhinoVLA采用了令牌高效的Qwen3-VL骨干网络和连续动作专家（Action Expert），减少了VLM侧的令牌和计算负担，同时保留了预训练的多模态能力。为了支持跨机器人学习，RhinoVLA进一步引入了一个统一接口，结合了视图注册表（View Registry）、72维物理状态-动作槽空间和机器人实例LoRA，使得异构的机器人观测和动作模式能够在共享策略下对齐。在部署方面，RhinoVLA通过硬件感知编译、混合精度执行和并行视觉编码进行了优化。实验表明，RhinoVLA在相似参数量下实现了与π0.5相当的下游性能，同时在辉羲R1上达到了11.69 Hz的端到端推理频率，满足了10 Hz实时闭环控制目标。该项目将在https://github.com/HuixiAI/RhinoVLA开源。

## Abstract
Vision-Language-Action (VLA) models have shown strong potential for robotic manipulation, but real-time deployment on edge hardware remains challenging. In this work, we identify VLM visual and context tokens as a major source of deployment latency: for GEMM-dominated projection operators, computation grows linearly with the number of input tokens when model dimensions are fixed. Motivated by this observation, we propose RhinoVLA, a deployment-oriented VLA model co-designed with the Huixi R1 edge SoC. RhinoVLA adopts a token-efficient Qwen3-VL backbone and a continuous Action Expert, reducing the VLM-side token and computation burden while preserving pretrained multimodal capability. To support cross-robot learning, RhinoVLA further introduces a unified interface that combines View Registry, 72D physical state-action slot space, and robotinstance LoRA, allowing heterogeneous robot observations and action schemas to be aligned under a shared policy. On the deployment side, RhinoVLA is optimized through hardware-aware compilation, mixed-precision execution, and parallel visual encoding. Experiments show that RhinoVLA achieves downstream performance comparable to π0.5 at a similar parameter scale, while reaching 11.69 Hz end-to-end inference on Huixi R1, meeting the 10 Hz real-time closedloop control target. The project will be open-sourced at https://github.com/HuixiAI/RhinoVLA.

## 1. 论文的核心问题与整体含义（研究动机和背景）
论文聚焦于**视觉-语言-动作（VLA）模型在边缘硬件上的实时闭环控制部署难题**。现有VLA系统（如π0.5）使用预训练的视觉-语言骨干和迭代动作生成模块，虽然性能强，但引入大量计算和内存流量，导致在搭载Jetson AGX Orin等边缘平台上的推理延迟显著（约858 ms），无法满足10 Hz的实时控制要求。作者通过分析发现，延迟的主要来源是**VLM骨干中的MLP投影算子**——由于它们是GEMM密集型，**计算量与输入token数线性相关**。因此，减少视觉和上下文token数成为降低延迟的关键。RhinoVLA以**算法-系统协同设计**为核心理念，旨在实现**与π0.5相当的操控性能，同时达到可部署在低功耗边缘SoC上的实时推理速度**。

## 2. 论文提出的方法论

### 核心思想
- 采用**token高效的VLM骨干**（Qwen3-VL），每张图像仅用64个合并token（对比PaliGemma的256个），从源头减少MLP GEMM计算量。
- 设计**统一的跨机器人训练框架**，通过视图注册表、72维物理状态-动作槽空间和实例LoRA适配器，使共享策略能对齐异构机器人的观测和动作模式。
- 在**部署侧**，通过**硬件感知编译、混合精度（W8A16）执行、并行视觉编码**优化Huixi R1芯片利用率。

### 关键技术细节
1. **架构**：采用“VLM骨干 + 连续Action Expert”双模块分离设计。VLM骨干为Qwen3-VL-2B（2.13B参数），Action Expert为0.40B参数，使用18层Qwen兼容Transformer，条件化在VLM的最后18层KV缓存、机器人状态/动作掩码、噪声动作块等上，预测72维统一动作空间上的流速度。
2. **View Registry**：为每个图像显式添加`[角色|模态]`标签（如`[head|rgb]`），解耦相机身份与数据集特定的顺序。
3. **72D统一槽空间**：将异构机器人的状态/动作映射到具有固定物理含义的72维向量（如D0–D6为臂关节弧度，D14–D15为夹爪闭合比等），并利用二进制掩码标记有效维度，避免零值监督。
4. **机器人实例LoRA**：在Action Expert的FFN层中插入低秩适配器，每个实例训练独立的LoRA，但共享主骨干和输出层，从而保留72D接口统一性，同时捕捉标定、控制等残留差异。
5. **训练策略**：预训练时冻结Qwen3-VL，联合优化VLM LoRA、共享Action Expert和实例LoRA。使用**掩码流匹配损失**，只对有效动作槽计算梯度。损失采用加权平衡不同动作组。
6. **高效部署**：
   - 编译优化：FlashAttention式分块、算子融合、细粒度调度。
   - 混合精度：INT8权重 + FP16激活，定制W8A16 GEMM内核，融合加载-反量化-计算，达到1.69倍加速。
   - 并行编码：将三路相机图像批处理编码，ViT延迟从34.52 ms降至24.31 ms。

## 6. 论文的主要结论与发现
1. **VLM的token数量是边缘VLA延迟的主要瓶颈**，减少token可有效降低MLP GEMM计算。
2. **Qwen3-VL作为token高效的骨干**，在保持多模态能力的同时，将视觉token降低至1/4，是实现实时推理的关键。
3. **统一接口（View Registry + 72D槽空间 + 实例LoRA）** 能成功对齐异构机器人数据，实现跨机器人策略学习，且实例LoRA能捕获本体残留差异，提升动作预测精度。
4. **经过硬件感知编译、W8A16混合精度、并行编码优化后**，RhinoVLA在辉羲R1上达到**11.69 Hz端到端推理**，满足10 Hz实时闭环控制需求。
5. **下游性能与π0.5相当**（LIBERO上90.0% vs π0.5 96.9%；真实任务接近或更优），证明了紧凑、可部署的VLA方案可以兼顾速度和性能。
