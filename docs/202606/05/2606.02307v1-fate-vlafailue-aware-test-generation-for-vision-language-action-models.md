## 摘要
视觉-语言-动作（VLA）模型越来越多地被用作通用机器人策略，但其评估仍主要依赖于随机采样任务场景的静态基准。在高维具身体空间中，故障稀疏且聚集，因此静态基准可能低估鲁棒性风险。我们将VLA评估重新定义为主动故障发现问题，并提出一种故障感知测试生成方法，该方法结合了多样性驱动的探索与从观测执行中学习的代理模型。该方法将测试导向高风险且多样化的场景区域。在四个最先进的VLA模型上，该方法发现了显著更多的故障（比所选基线最多提升29.7%），同时揭示了更多样化的故障模式。这意味着，例如，在GR00T-N1.6的情况下，成功率从64.4%下降到34.7%。更广泛地说，我们的发现呼吁VLA评估的转变：从固定任务套件上的被动测量转向自适应的、寻求故障的测试生成，从而在部署前暴露模型弱点的结构。

## Abstract
Vision-Language-Action (VLA) models are increasingly used as generalist robot policies, yet their evaluation still relies largely on static benchmarks that randomly sample task scenes. In high-dimensional embodied spaces, failures are sparse and clustered, so static benchmarking can underestimate robustness risks. We reframe VLA evaluation as an active failure-discovery problem and propose a failure-aware test-generation approach that combines diversity-driven exploration with surrogate models learned from observed executions. The method steers testing toward high-risk yet diverse scene regions. Across four state-of-the-art VLA models, it uncovers substantially more failures (up to +29.7 % over selected baselines) while revealing more diverse failure modes. This mean that, for instance, in the case of GR00T-N1.6, success rate dropped from 64.4% to 34.7%. More broadly, our findings call for a shift in VLA evaluation: from passive measurement on fixed task suites to adaptive, failure-seeking test generation that exposes the structure of model weaknesses before deployment.

### 1. 论文的核心问题与整体含义（研究动机和背景）
当前视觉-语言-动作（VLA）模型被广泛用作通用机器人策略，但其评估仍严重依赖静态基准，通过随机采样任务场景来计算成功率。然而在 **高维具身空间** 中，故障是 **稀疏且聚类** 的（即集中在特定场景区域），随机采样会导致对鲁棒性风险的 **严重低估**，且无法系统揭示模型的弱点结构。  
**核心动机**：将 VLA 评估从“被动测量”重新定义为 **主动故障发现问题**——自适应地学习故障分布，并生成 **既高风险又多样化** 的测试场景，以暴露更广更深的模型脆弱性。

### 2. 论文提出的方法论
#### 核心思想
- 将 **自适应随机测试 (ART)** 与 **机器学习代理模型** 结合。  
- 在保持输入场景多样性的同时，优先选择 **代理模型预测为故障** 的场景。  
- 代理模型基于 **已执行场景的结果**（成功/失败）进行迭代训练，逐步学习故障区域结构。

#### 关键技术细节
- **场景表示**：每个测试场景由物体类型、位置 (x, y) 和朝向组成，编码为连续数值向量。  
- **距离度量**：欧几里得距离，用于 FSCS-ART 中候选与已执行场景的最小距离计算。  
- **代理模型**：  
  - 决策树 (DT)：最大深度 8  
  - 随机森林 (RF)：150 棵树，最大深度 12，最小分裂样本 3  
- **自适应权重 α**：`α = min(0.85, i/N)`，随测试轮数 i 线性增加，前半段侧重探索，后半段侧重利用。

#### 两种算法流程（文字说明）
**算法1：混合排序代理辅助 FSCS-ART**  
1. 初始无代理模型时，使用 FSCS-ART 选择与已执行场景距离最大的候选（纯探索）。  
2. 收集至少两类样本（成功和失败）后，训练 DT 代理模型。  
3. 后续迭代：先按与已执行场景的最小距离对候选排序，再按序检查代理模型预测；若预测为故障，则优先选择该候选；否则选择多样性最高（排序第一）的候选。  
4. 执行后更新记录并重新训练代理模型。

**算法2：混合自适应 RF 引导 FSCS-ART**  
1. 初始阶段同算法1的纯探索。  
2. 代理模型为随机森林 (RF)，输出故障概率。  
3. 后续迭代：候选集从已发现的故障区域附近（局部）与全局混合生成；对每个候选计算 **综合得分 = α × 故障概率 + (1-α) × 归一化最小距离**；选择得分最高的候选执行。  
4. 失败场景存入单独故障档案，代理模型每轮重训。
