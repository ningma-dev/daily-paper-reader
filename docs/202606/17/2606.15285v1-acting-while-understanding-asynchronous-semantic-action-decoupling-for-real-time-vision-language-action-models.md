## 摘要
视觉-语言-动作模型（VLA）在机器人操作中展示了强大的任务理解和泛化能力，然而全模型推理的高计算成本限制了其在低延迟、高频率闭环控制中的部署。我们提出了一种异步语义-动作解耦框架，沿着现有VLA的内部语义-动作接口，将语义理解与动作生成分离，无需重新设计视觉-语言骨干网络或引入外部规划器。低频理解模块异步更新可复用的语义条件，而高频动作模块持续输出控制动作，无需重复调用完整模型。为缓解过时语义与当前执行状态之间的时间错配，我们进一步引入历史动作条件化与时间错位训练，提供短时域执行上下文，并在过时语义条件下提升反馈控制的鲁棒性。在LIBERO上使用π₀.₅和UniVLA进行的实验，以及使用UniVLA的真实机器人部署表明，所提框架实现了高达35.6 Hz的服务端动作模块推理吞吐量，并提供了一条低侵入性路径，实现无需以控制频率运行全VLA推理的高频闭环控制。

## Abstract
Vision-Language-Action models (VLAs) have demonstrated strong task understanding and generalization in robotic manipulation, yet the high computational cost of full-model inference limits their deployment in low-latency, high-frequency closed-loop control. We propose an asynchronous semantic-action decoupling framework that separates semantic understanding from action generation along the internal semantic-action interface of existing VLAs, without redesigning the vision-language backbone or introducing an external planner. A low-frequency understanding module asynchronously updates reusable semantic conditions, while a high-frequency action module continuously outputs control actions without repeatedly invoking the full model. To mitigate the temporal mismatch between stale semantics and the current execution state, we further introduce historical action conditioning and time-misalignment training, which provide short-horizon execution context and improve feedback control robustness under stale semantic conditions. Experiments on LIBERO with $π_{0.5}$ and UniVLA, together with real-robot deployment using UniVLA, show that the proposed framework achieves up to 35.6 Hz server-side action-module inference throughput and offers a low-intrusion path to high-frequency closed-loop control without running full VLA inference at control rate.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **背景**：视觉-语言-动作模型（VLA）在机器人操作中展现出强大的任务理解与泛化能力，但全模型推理计算成本极高（尤其是视觉-语言骨干网络），导致推理频率远低于机器人控制频率（通常需要 20 Hz 以上）。
- **问题**：现有 VLA 普遍采用动作分块（action chunking）策略来缓解频率不匹配，但这本质上是同步耦合——语义理解与动作生成必须在同一时间步完成，且整个分块执行期间语义推断保持静态。当环境变化或任务阶段推进时，机器人仍依赖过时的语义判断，导致控制滞后或错误。
- **核心挑战**：如何在保持高频状态反馈控制的同时，避免运行全模型推理；特别是当复用过时（stale）的语义条件时，如何维持稳定控制。

### 2. 论文提出的方法论：核心思想、关键技术细节

#### 核心思想
沿着 VLA 内部的“语义-动作接口”（如 π₀.₅ 的 KV-cache 或 UniVLA 的最后层隐藏状态），将原始同步模型解耦为两个异步角色：
- **低频理解模块**：以约 3–5 Hz 频率处理视觉观测与语言指令，更新可复用的语义缓存。
- **高频动作模块**：以控制周期频率（20 Hz）运行，仅使用最新缓存语义、当前机器人状态和最近动作历史，连续输出动作。

#### 关键技术细节
1. **异步语义-动作解耦**：
   - 定义语义条件 \( z_t = U_\theta(o_t, l, \tilde{s}_t) \)，动作生成 \( \hat{A}_t = G_\phi(z_{\tau_k}, s_t) \)，其中 \( \tau_k \leq t \) 为最新语义更新时间。
   - 动作模块不等待完整推理，直接读取已提交的语义缓存。

2. **历史动作条件化**：
   - 引入最近 m 步动作历史 \( h_t = [a_{t-m}, \dots, a_{t-1}] \)，作为动作模块的额外输入：\( \hat{A}_t = G_\phi(z_{\tau_k}, s_t, h_t) \)。
   - 历史动作提供短时域执行上下文，帮助克服语义过时带来的进度模糊性。

3. **时间错位训练（Time-Misalignment Training）**：
   - 在训练时模拟部署中的语义延迟：均匀采样延迟步数 \( \Delta \in \{0, \dots, H-1\} \)（H 为动作块长度），训练目标为 \( \hat{A}_t = G_\phi(z_{t-\Delta}, s_t, h_t) \)，损失函数 \( \mathcal{L} = \mathbb{E}_{t, \Delta} [\ell_{\text{base}}(G_\phi(z_{t-\Delta}, s_t, h_t), A_t)] \)。
   - 训练时历史动作添加小高斯噪声，推理时使用实际执行的动作。

4. **推理流程**：
   - 理解模块异步更新语义缓存（约 3–5 Hz）。
   - 动作模块每控制步执行一次：使用最新缓存语义、当前状态、当前动作历史，重新规划一个完整动作块，并采用 ACT 风格的指数加权时序集成（decay=0.1）融合重叠预测，输出最终动作。

### 6. 论文的主要结论与发现

- **主要结论**：异步语义-动作解耦框架在保持强语义指导的同时，将动作模块推理吞吐量提升至 35.6 Hz（UniVLA-Async），远高于同步的 3.4 Hz，且无需运行全模型推理。

- **真实机器人**：UniVLA-Async 平均成功率 93.3%，高于同步 UniVLA 的 78.3%，且 SO100 上完成时间从 17.4s 降至 13.1s。
- **消融证实**：单纯缓存复用导致性能下降（从 92.0% 到 85.2%）；结合历史动作和延迟训练可恢复至 91.2%，接近同步水平。其中延迟训练收益更大，表明时序错位是主要难点。

### 7. 优点

1. **低侵入性**：无需修改 VLA 骨干网络，仅沿着内部语义-动作接口解耦，改动集中在动作生成侧。
2. **无需外部规划器**：利用 VLA 自身内部状态（隐藏状态或 KV-cache）作为可复用语义，保持端到端特性。
3. **双重鲁棒机制**：历史动作提供局部进度信号，延迟训练显式模拟部署时序错位，两者互补，有效缓解过时语义问题。
4. **普适性验证**：在两个不同骨干（π₀.₅、UniVLA）和两种真实平台（Kinova、SO100）上都取得积极结果，说明框架具有跨模型、跨实体适用性。
5. **清晰的消融实验**：严谨地分离了每个组件（缓存、历史、延迟训练）的贡献，支撑核心论点。

### 8. 不足与局限

1. **依赖内部接口**：要求 VLA 存在可复用的中间语义表示（隐藏状态或 KV-cache），因此框架不能直接应用于那些没有清晰内部接口的模型。
2. **语义过时风险**：尽管历史动作和延迟训练缓解了问题，但若语义缓存年龄过大或环境发生剧烈变化（如突然场景切换），动作模块仍可能被误导。当前固定刷新频率并非最优。
3. **实验覆盖有限**：仅测试了单臂操作任务，未包含快速动态操作、移动操作、复杂接触交互等场景。
4. **真实机器人 π₀.₅ 缺失**：未在 π₀.₅ 上进行真实部署验证，其服务端吞吐量（19.7 Hz）加上通信开销后不足以稳定跟踪 20 Hz 控制循环，限制了结论的通用性。
5. **未做超参数敏感性分析**：未探讨历史窗口长度、刷新频率、时序集成权重等对性能的影响。
6. **潜在偏差风险**：真实机器人任务数量较少（每平台 3 个），且使用手动重置与标准种子协议，可能不足以反映真实世界变异性。同时，消融实验仅在一个套件（LIBERO-Long）上进行，未在其余三个套件中验证。
