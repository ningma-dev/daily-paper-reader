## 摘要
世界动作模型（WAMs）通过将未来预测作为额外的训练目标，扩展了机器人策略学习，鼓励策略在其表示中编码与任务相关的时间结构。当前的WAMs通常依赖大型生成式架构，这些架构导致高训练成本和推理延迟，难以作为高效的闭环策略部署。我们提出了Light-WAM，一种用于高效机器人操作的轻量级世界动作模型。具体而言，它采用紧凑的视频骨干网络，并在降采样的潜在空间中进行未来视频监督，从而降低视频协同训练的成本，同时保留其对表示学习的益处。对于动作预测，Light-WAM引入了StateFusionActionExpert，它从多个骨干层读取适配状态，通过习得查询池化进行融合，并在单次前向传播中直接预测动作块。该设计在视频骨干表示和机器人动作之间提供了高效接口，避免了对重型生成式动作专家的需求。实验表明，Light-WAM在LIBERO上保持了强大的性能，并在RoboTwin 2.0上实现了可用的多任务性能，同时仅使用0.44B可训练参数。此外，它实现了72.03ms的推理延迟，峰值GPU内存为4.1GiB，并提高了训练吞吐量。

## Abstract
World Action Models (WAMs) extend robot policy learning by incorporating future prediction as an additional training objective, encouraging the policy to encode task-relevant temporal structure in its representations. Current WAMs often rely on large-scale generative architectures that incur high training costs and inference latency, making them difficult to deploy as efficient closed-loop policies. We propose Light-WAM, a lightweight World Action Model for efficient robot manipulation. Specifically, it is built with a compact video backbone and performs future-video supervision in a downsampled latent space, reducing the cost of video co-training while retaining its benefits for representation learning. For action prediction, Light-WAM introduces the StateFusionActionExpert, which reads adapted states from multiple backbone layers, fuses them through learned-query pooling, and directly predicts action chunks in a single forward pass. This design provides an efficient interface between video backbone representations and robot actions, avoiding the need for heavy generative action experts. Experiments demonstrate that Light-WAM maintains strong performance on LIBERO and achieves usable multi-task performance on RoboTwin 2.0, while using only 0.44B trainable parameters. It also achieves 72.03ms inference latency with 4.1GiB peak GPU memory and improved training throughput.

- **核心问题**：能否在保留未来视频预测的训练收益的同时，大幅降低模型参数量和计算开销，使WAMs变得轻量、高效、可用？
- **整体含义**：Light-WAM探索了“训练时用低分辨率潜空间未来视频监督，推理时不生成未来视频”的路径，证明可以通过紧凑的视频骨干和直接的动作解码器实现接近甚至超越大型WAMs的性能，为实际机器人部署提供更实用的方案。

## 2. 方法论：核心思想、关键技术细节

### 2.1 核心思想
- 使用预训练的紧凑视频骨干（Wan2.1-T2V-1.3B）作为特征提取器，冻结大部分权重，仅通过轻量模块（LoRA + 稀疏WAM适配器）进行任务适配。
- 未来视频监督只在训练时进行，且在**降采样的潜空间**中执行（2×空间下采样），大幅降低训练时的token数。
- 推理时无需生成未来视频，而是通过设计的**StateFusionActionExpert**模块，从多层级骨干状态中直接预测动作块，一步到位，避免复杂的迭代式生成过程。

### 2.2 关键技术细节

#### 视频骨干适配
- 使用 **Wan2.1-T2V-1.3B** 作为视频骨干，保留其强大的视频先验。
- 冻结原骨干权重，在所有Transformer块的注意力与前馈投影上应用**LoRA低秩适配**。
- 在稀疏的深度层级（默认{8, 16, 24}）插入**WAM适配器**（轻量瓶颈MLP），为动作解码器提供多层级状态输出。

#### 高效潜空间视频协同训练
- 将视频潜码 `z_vid` 进行2×空间下采样得到 `\bar{z}_vid`，然后对下采样后的潜码进行流匹配（flow-matching）扰动，计算预测与目标间的L2损失。
- 动作分支使用原始分辨率的当前观测潜码 `z_act`，保持细节。

#### StateFusionActionExpert
- 对每个选定骨干层，使用一组可学习查询（Nq=16）通过多头注意力从该层的视频token中压缩出层级状态 `s_ℓ`。
- 将各层状态拼接、投影、融合得到固定维度的融合状态 `h`。
- 利用步嵌入（step embedding）为时间步解码，输出动作块。
- 整个动作解码只需单次前向传播，无迭代去噪，比基于DiT的动作头轻量很多。

#### 训练目标
- 联合损失：`L = L_video + λ L_action`，其中L_video为潜空间流匹配损失，L_action为动作块回归损失。

## 6. 主要结论与发现

1. **Light-WAM在LIBERO上达到97.2%平均成功率**，在无具身预训练方法中排名第一，整体排名第三（仅次于Motus和LingBot-VA），证明轻量级设计并未牺牲标准基准的性能。

3. **效率优势显著**：训练吞吐量提升4.25×，推理延迟降低至Fast-WAM的1/5.6，峰值GPU内存降至4.1GiB（Fast-WAM为12.7GiB），GPU内存需求低至43.1GiB（训练时）。
4. **消融验证**：
   - 未来视频监督分辨率越高（1×）性能越好，但训练成本剧增；2×下采样是合理的折中。

   - 16个查询优于8个，证明足够容量对保留视觉细节重要。
5. **真实世界评估**：在三项双机械臂任务上，Light-WAM平均成功率高于π0.5（例如“抓取物体并放置”93% vs 67%），说明轻量模型在实际场景中也可获得良好表现。

## 7. 优点

- **设计高效**：同时降低训练和推理成本，实现“一个模型两用”（视频协同训练+直接动作预测），无需测试时生成未来视频。
- **StateFusionActionExpert设计巧妙**：利用可学习查询从多层骨干压缩信息，避免高维token处理；单次前向解码动作，比扩散式动作头更轻快。
- **训练时潜空间降采样的创新**：2×下采样减少了未来视频监督的token数量，同时保留了足够的时间结构信息。
- **充分消融**：对关键设计逐一验证，决策有数据支撑。
- **硬件友好**：推理峰内存仅4.1GiB，延迟72ms，非常适合实时闭环控制。
- **可复现**：开源代码。

## 8. 不足与局限

1. **多任务复杂场景扩展性有限**：在RoboTwin 2.0的50任务设置中，Light-WAM大幅低于Fast-WAM（76.4% vs 91.9%），说明轻量模型在更大、更复杂的多任务学习场景中仍面临容量瓶颈。
2. **缺乏泛化与鲁棒性测试**：未在LIBERO-Plus等专门评估分布外泛化和鲁棒性的基准上进行实验，也未涉及域随机化、数据增强等训练策略。作者在结论中承认这一局限。
3. **未来视频监督质量有限**：由于在潜空间降采样，预测的未来视频模糊（定性图也显示平滑但缺乏细节），可能限制其在需要精细运动预测的任务上的潜力。
4. **依赖特定预训练骨干**：Wan2.1-T2V-1.3B的选择可能不是最优或唯一选择，尚未探索不同骨干的通用性。
5. **仅评估双机械臂任务**：真实世界实验仅在一个特定平台（IMETA Y1）的3个任务上完成，泛化性验证不足。
6. **动作解码未考虑历史上下文**：模型每次只依赖当前单帧观测，未利用过去帧或显式状态记忆，这在某些需要记忆的任务中可能成为瓶颈。
