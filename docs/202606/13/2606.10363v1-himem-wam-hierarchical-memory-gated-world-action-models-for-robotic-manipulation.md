## 摘要
世界动作模型（WAMs）已成为具身智能的一种新的强大范式，通过学习与动作相关的视觉动态，显著提升了泛化性和鲁棒性。然而，现有的WAMs在长期机器人操作中仍难以处理与任务相关的记忆。为此，我们提出HiMem-WAM，一种分层记忆门控WAM，它整合了以运动为中心的潜在动作、高层技能潜在变量以及边界触发的记忆更新。具体而言，我们开发了一个分层潜在动作框架，联合学习低层运动和高层技能潜在变量，提供了结构化的时间抽象。同时，一个边界感知记忆门在预测的技能转换处写入紧凑的任务状态，使得无需在测试时生成未来视频或光流估计即可进行因果推理。在LIBERO、LIBERO-PLUS、RMBench以及真实世界任务上的评估表明，HiMem-WAM的分层潜在变量提升了部署扰动下的鲁棒性，而记忆模块显著有利于依赖记忆的长期操作。

## Abstract
World Action Models (WAMs) have emerged as a new powerful paradigm for embodied intelligence, learning action-relevant visual dynamics that significantly enhance generalization and robustness. However, existing WAMs still struggle with task-relevant memory in long-horizon robotic manipulation. To address this, we present HiMem-WAM, a Hierarchical Memory-Gated WAM that integrates motion-centric latent actions, high-level skill latents, and boundary-triggered memory updates. Specifically, we develop a hierarchical latent action framework that jointly learns low-level motion and high-level skill latents, providing structured temporal abstraction. Meanwhile, a boundary-aware memory gate writes compact task states at predicted skill transitions, enabling causal inference without test-time generation of future video or optical flow estimation. Evaluated on LIBERO, LIBERO-PLUS, RMBench and real-world tasks, HiMem-WAM shows that hierarchical latents improve robustness under deployment perturbations, and the memory module substantially benefits memory-dependent long-horizon manipulation.

## 2. 论文提出的方法论

### 核心思想
将长时程操作分解为：**技能选择（高层）** → **运动展开（低层）** → **动作解码**，并通过边界触发的记忆更新来维持任务历史，实现完全的因果推理（无需测试时生成未来视频或光流）。

### 关键技术细节
- **低层潜在动作 tokenizer**：利用多视角光流（DPFlow）和变分编码器，将短时上下文（当前帧、下一帧、光流、语言、本体感觉）编码为低层潜在动作 \(z_l\)，训练时通过重建光流和（若可用）对齐真实动作来学习。训练后冻结，离线提取所有轨迹的 \(Z^l\) 序列。
- **高层技能潜在变量**：通过**层次化分块机制**自动发现技能边界。对低层序列 \(Z^l\)，逐级编码、计算边界得分（基于相邻 token 的相似度），然后划分可变长度片段，通过注意力池化得到高层技能 latent \(z^h\)。边界标签 \( \bar{b}_t \) 作为监督。
- **记忆门控模块**：
  - 状态编码 \(x_t = E_\theta(o_t, p_t, \ell)\)
  - **读门**：对外部记忆库 \(M_t\) 进行注意力检索，得到记忆上下文 \(c_m^t\)，通过 sigmoid 门控 \(\alpha_r^t\) 控制融合量，形成记忆增强状态 \(\tilde{x}_t\)。
  - **规划器**：基于 Qwen3-VL-4B-Instruct，预测当前技能 latent \(\hat{z}^h_t\) 和边界得分 \(\hat{b}_t\)。
  - **执行器**：将技能展开为低层动作 chunk \(\hat{Z}^l_{t:t+K-1}\)。
  - **动作解码器**：将低层动作映射到实际控制信号 \(\hat{a}_{t:t+K-1}\)。
  - **写门**：用 sigmoid 门控 \(\alpha_w^t\) 决定是否写入新记忆 token（当 \(\alpha_w^t > \eta\) 时）。写入内容由当前状态、预测技能和池化后的低层动作 chunk 拼接构成。写门受边界标签监督，确保只在有意义的状态转换处写入。
- **训练三阶段**：
  - **Stage I**：训练低层 tokenizer（用光流和可选动作标签）。
  - **Stage II**：冻结 tokenizer，进行技能发现和规划器/执行器预训练（无外部记忆），损失包括技能 MSE、动作 chunk MSE 和边界 BCE。
  - **Stage III**：激活记忆门控模块，在带动作标签的演示上全模型微调，使用完整损失 \(L_{ft} = L_{act} + \alpha_h L_{plan} + \alpha_l L_{exec} + \alpha_b L_{bd} + \alpha_m L_{gate}\)，其中门控损失包含写门 BCE 和读/写门的稀疏正则化。

### 推理流程
纯因果：仅需当前 RGB、本体感觉、指令和记忆库 → 读门检索 → 规划器预测技能和边界 → 执行器产生动作 chunk → 解码输出 → 若写门超过阈值则写入新记忆。无未来生成。
