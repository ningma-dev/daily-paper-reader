## 摘要
机器人技术面临数据稀缺的根本性挑战。与语言或视觉研究不同，机器人操作领域没有互联网规模的数据集。一个有前景的前进方向是利用以自我为中心的人类数据，这类数据更容易收集，具有更广的广度，且规模更大。为此，我们研究了在配备灵巧五指手的人类和人形实体之间进行学习的关键设计选择，以$π_{0.5}$模型为基础。我们的结果表明，人类数据使机器人无需相应的机器人数据即可学习新的任务语义并将现有技能组合成新颖的行为。论文网站：https://egopipaper.github.io/

## Abstract
Robotics faces a fundamental challenge of data scarcity. Unlike language or vision research, there is no internet-scale dataset for robotic manipulation. A promising path forward is to leverage egocentric human data, which can be collected more easily, with greater breadth, and at a larger scale. Towards this end, we investigate key design choices for learning across human and humanoid embodiments equipped with dexterous five-finger hands, using the $π_{0.5}$ model as a foundation. Our results show that human data enables robots to learn new task semantics and compose existing skills into novel behaviors without corresponding robot data. - **研究动机**：机器人操作面临严重的数据稀缺问题，缺乏互联网规模的数据集。相比之下，第一人称（ego-centric）人类数据（如通过Apple Vision Pro、Meta Ray-Ban等设备收集）更容易大规模获取。

## 2. 方法论
- **基础模型**：采用π₀.5（flow-matching VLA），预训练于gripper机器人的低维动作（最大32维），需适配灵巧手的高维动作（每手29维，双手共58维）。
- **关键技术细节**：
  - **动作对齐（Action Alignment）**：将人类手部关键点（MANO标准20个关键点）映射到机器人手关节空间，采用每关节偏移量和缩放因子的线性映射，避免逆运动学带来的自碰撞问题。
  - **动作交错（Interleaved Action Tokens）**：为解决动作维度超出π₀.5容量，将左右手动作分布到两个独立token中（每个token最大32维），预测时按`{a_L, a_R, ...}`交错排列，虽然有效horizon减半，但可保持预训练权重不变。
  - **视觉对齐**：在输入图像上绘制颜色编码的骨架线（按手指对应关系），且按深度顺序遮挡绘制，使模型明确跨体现的手指对应关系。
  - **子任务预测（Subtask Prediction）**：作为辅助损失，训练VLM部分输出子任务字符串（如“pick tomato”），引导模型“思考后再行动”，对技能链式任务至关重要。
- **训练细节**：
  - 人类:机器人数据比例=50%:50%；机器人手腕相机以40%概率dropout以对齐人类缺少手腕相机的设置。
  - 优化器AdamW，余弦学习率调度，weight decay=0，gradient clip=1.0，batch size=128，训练步数5000-10000。

## 6. 主要结论与发现
- **简单co-training** 在番茄分拣（92%）和包装（90%）任务上有效，但在装箱任务上仅27%（常发生先抓积木或双手碰撞）。

- **骨架线**在所有任务中几乎无提升效果。
- **手型差异**：Inspire手（更接近人手尺寸）在装箱任务中优于Tesollo手（67% vs 93%），后者因尺寸过大导致视觉差异和状态敏感。
- **手腕相机**在测试时不可或缺，移除后番茄分拣抓取不稳定。
- **关键结论**：人类数据可成功传递高层任务语义（排序、链式、规则），无需对应机器人演示；子任务预测是解锁复杂技能组合的有效手段。
