## 摘要
视觉-语言-动作（VLA）模型建立在这样一个前提上：来自预训练语言或视觉-语言骨干网络的语义理解应指导机器人动作预测。然而，机器人微调被优化为对任务特定动作分布的模仿，并且许多评估可以通过视觉或指令-动作捷径来解决。我们引入了RoboSemanticBench（RSB），这是一个用于诊断动作预测中语义基础的具身基准：经过后训练的VLA模型能否利用复杂的指令语义来选择并操控正确的物理目标。在每个回合中，机器人接收一个多项选择的数学或常识问题，观察候选答案块，并且必须抓取对应正确答案的块。RSB涵盖了受控算术、小学水平的数学理解以及常识或事实理解，分为四选一和十选一套餐。在代表性VLA模型中，我们发现许多策略学会了抓取候选块，但在控制抓取成功后发现以接近随机或低于随机的速率选择语义正确的块，这揭示了骨干级别的语义能力与动作预测之间存在持续差距。

## Abstract
Vision-language-action (VLA) models are built on the premise that semantic understanding from pretrained language or vision-language backbones should guide robot action prediction. Yet robot fine-tuning is optimized as imitation over task-specific action distributions, and many evaluations can be solved through visual or instruction-action shortcuts. We introduce RoboSemanticBench (RSB), an embodied benchmark for diagnosing semantic grounding in action prediction: whether post-trained VLA models can use complex instruction semantics to select and manipulate the correct physical target. In each episode, a robot receives a multiple-choice math or general-knowledge question, observes candidate answer blocks, and must grasp the block corresponding to the correct answer. RSB covers controlled arithmetic, grade-school mathematical understanding, and commonsense or factual understanding under four-choice and ten-choice suites. Across representative VLA models, we find that many policies learn to grasp candidate blocks but select the semantically correct block at near-random or below-random rates after controlling for grasp success, revealing a persistent gap between backbone-level semantic competence and action prediction.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：视觉-语言-动作（VLA）模型依赖预训练骨干网络的语义理解来指导机器人动作预测，但实际微调过程可能使模型学会视觉或指令-动作捷径，而非真正理解指令语义。论文旨在诊断这种“语义基础”（semantic grounding）在动作预测中是否真实存在。
- **整体含义**：当前VLA模型即使在大规模预训练后，其动作决策依然很少受指令语义的驱动，导致在需要复杂语义理解的任务中表现近于随机。这一缺陷限制了VLA在真实世界复杂指令场景中的可信应用。

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：设计一个可控的具身问答基准RoboSemanticBench（RSB），要求机器人根据多项选择数学或常识问题从一组候选块中抓取对应正确答案的块。通过固定操作原语（抓取并放置），仅改变语义内容，分离低层抓取能力和高层语义目标选择能力。
- **关键技术细节**：
  - **任务构建**：包含三个语义子集：RSB-Math（程序化算术题）、RSB-HardMath（GSM8K改编的数学应用题）、RSB-General（MMLU风格的常识/事实题）。
  - **选择套件**：每个子集有4选和10选两种规模（10选时跳过J以避免视觉混淆）。块布局、选项-字母映射每个回合随机化，位置与颜色不能作为捷径。
  - **指令生成**：使用模板暴露问题和映射，但从不包含正确答案，防止标签泄露。
  - **仿真环境**：基于物理引擎的桌面模拟器，使用Aloha双机械臂、多视角RGB相机，专家演示通过运动规划生成抓取轨迹。
  - **关键指标**：
    - **任务成功率（TSR）**：正确抓取答案块的比例。
    - **抓取成功率（GSR）**：抓取任意候选块的比例。
    - **归一化语义基础得分（nSG）**：剥离低层抓取能力后衡量语义目标选择，公式为 `nSG = (TSR/GSR - 1/N) / (1 - 1/N)`，0对应随机选择，1对应完美选择。

### 3. 实验设计：数据集 / 场景 / Benchmark / 对比方法

- **数据集**：
  - RSB-Math：程序生成500个训练问题（4选和10选共用）。
  - RSB-HardMath：来自GSM8K训练集7,473个问题。
  - RSB-General：采样自MMLU风格问题，4选4,500个，10选10,000个。
  - 评估时使用与训练不重叠的问题集，每套评估500回合。
- **场景**：桌面抓取任务，机器人需将正确选项块放到灰色答案区。同时提供了一个“Beyond Blocks”变体，用日常物品替代字母块，验证通用性。
- **Benchmark**：RSB自身即为诊断基准，无外部对比基准。
- **对比方法**：9种代表性VLA模型——OpenVLA-OFT、GO1、DexVLA、TinyVLA、PD-VLA、π0、π0.5、GR00T N1.7、QwenGR00T。所有模型在相同专家演示数据上微调，训练步数×批次大小保持一致（100,000步×64）。

### 4. 资源与算力

- 论文明确说明所有模型在8块NVIDIA H100 GPU上训练，使用DeepSpeed ZeRO-2优化，训练步数均为100,000步，全局批次大小64（cotrain实验中VQA流批次为32）。但未报告具体训练时长（如小时数）。

### 5. 实验数量与充分性

- **实验数量**：主实验包含9个模型×6个套件=54组TSR/GSR/nSG数据。另有两组失败探索实验：ReasoningVLA和VLA cotrain，以及Beyond Blocks场景实验。误差分析对RSB-Math-4子集进行了细粒度类别统计。
- **充分性与公平性**：
  - 保持相同训练预算（步数×批次），使用官方代码库默认配置，尽量公平。
  - 指标设计（GSR/TSR分离、nSG归一化）消除了低层抓取能力的干扰，客观度量语义基础。
  - 但实验局限于单一仿真环境和一种桌面抓取任务，未涉及真实机器人；只测试了数学和常识问答，未覆盖更开放或因果推理指令；模型超参数未逐一优化，可能未充分发挥每个模型潜力。

### 6. 论文的主要结论与发现

- **主要结论**：当前VLA模型在动作预测中普遍缺乏语义基础。即使能够成功抓取任意候选块（高GSR），选择正确答案块的比例仍接近或低于随机（nSG≤0）。唯一例外是π0.5，取得正nSG（平均5.2%），但仍远非可靠。
- **失败探索结论**：
  - 引入CoT的ReasoningVLA仅小幅提升TSR（平均10.7%→16.0%），但大部分失败发生在CoT正确而动作仍抓错块，说明语义决策未能有效路由到动作路径。
  - VLA与VQA联合训练反而导致TSR下降（10.7%→8.2%），表明语言监督可能与动作学习冲突。
- **误差分析**：在抓取成功但任务失败的情况下，95%以上是由于选择了错误目标而非放置失败，证实问题根源于语义目标选择。

### 7. 优点：方法或实验设计上的亮点

- **诊断性设计**：将低层操控能力与高层语义选择解耦，通过GSR–TSR差距和nSG指标直观揭露语义基础缺失，避免传统任务成功率的混淆。
- **控制变量严格**：固定抓取原语、随机化布局和映射、不显示正确答案，使唯一困难来自语义理解。
- **语义覆盖广**：包含算术、多步数学应用题和常识问答，覆盖不同复杂度。
- **可复现性**：开源了全部训练数据、仿真评估代码和模型微调配置。
- **消融与干预**：不仅benchmark现有模型，还尝试了两种直观干预（CoT和cotrain），并揭示其不足，为未来研究提供方向。

### 8. 不足与局限

- **任务简化**：仅涉及单一抓取-放置原语，未覆盖多步操作、导航或其他复杂动作，语义基础可能在更丰富任务中表现不同。
- **仿真与现实差距**：实验全在模拟器中进行，不存在真实光照、纹理、传感器噪声等，模拟结果向真实迁移性存疑。
- **语义范围有限**：只测试数学和常识问答，未涉及模糊指令、多义性、空间推理或社交情境等更接近真实人机交互的指令。
- **模型微调深度不足**：所有模型采用统一训练预算，未针对每个模型进行超参数调优，可能低估某些模型的能力（如π0.5的更高GSR可能受益于其预训练中的子任务标注）。
- **指标局限性**：nSG依赖于GSR非零，当GSR很低时，nSG不可靠或缺乏统计意义（如DexVLA在10选套件中GSR仅17.5%）。
- **潜在数据偏差**：RSB-HardMath和RSB-General的训练数据来自GSM8K/MMLU，这些基准本身可能含有数据覆盖偏差或测试样例与训练分布不一致的问题。
