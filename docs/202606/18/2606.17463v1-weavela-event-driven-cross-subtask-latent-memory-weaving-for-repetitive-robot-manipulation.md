## 摘要
视觉-语言-动作（VLA）策略已实现显著的单步操作，但在每个阶段依赖于前一步完成之处仍然脆弱。核心问题在于结构性的：短窗口VLA缺乏跨子任务边界进行信息路由的显式通道，而现有的记忆增强变体要么在每一帧写入，从演示时间阶段检索，或在子目标事件触发时，但没有执行显式的子任务到子任务的交接至动作专家。我们将子目标完成事件识别为跨子任务内存交接的自然时间单位，并提出WeaveLA（为视觉-语言-动作策略编织潜在记忆），一种跨子任务记忆接口，在冻结的VLA骨干网络之上，通过查询驱动的注意力池化将每个已完成段压缩为潜在标记，并直接将它们路由到下一个子任务的动作生成路径中。这种事件触发的、动作侧的设计保留了基础策略的短窗口接口，同时增加了轻量级的跨子任务通道。通过在RoboMME上使用π0.5骨干网络进行分层评估，WeaveLA的增益恰好出现在需要该通道的地方：在最困难的重复切片（SwingXtimes，N=3）上，成功率从0%上升至47.8%，而单次执行场景保持不变。每集配对分析确认增益仅限于因果结构需要跨子任务信息的任务。

## Abstract
Vision-Language-Action (VLA) policies have achieved remarkable single-step manipulation, yet they remain brittle precisely where each stage depends on what was just completed. The core issue is structural: short-window VLAs lack an explicit channel for rouxting information across sub-task boundaries, and existing memory-augmented variants either write at every frame, retrieve from demonstration-time stages, or fire at sub-goal events without performing an explicit sub-task-to-sub-task hand-off into the action expert. We identify the sub-goal completion event as the natural temporal unit for cross-subtask memory hand-off, and present WeaveLA (Weave Latent memory for Vision-Language-Action policies), a cross-subtask memory interface that, on top of a frozen VLA backbone, compresses each completed segment into latent tokens via query-driven attention pooling and routes them directly into the action-generation path of the next sub-task. This event-triggered, action-side design preserves the base policy's short-window interface while adding a lightweight cross-subtask channel. Through stratified evaluation on RoboMME with a $π_{0.5}$ backbone, WeaveLA's gains land exactly where the channel is needed: on the hardest repetition slice (SwingXtimes, $N{=}3$), success rises from $0\%$ to $47.8\%$, while single-execution episodes remain unchanged. Per-episode paired analysis confirms the gains are confined to tasks whose causal structure requires cross-subtask information.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：现有视觉-语言-动作（VLA）策略在**重复性机器人操作**中表现脆弱，原因在于它们缺乏一个**显式的跨子任务信息通道**。短窗口VLA依赖当前观测，而重复任务（如多次抓取、来回摆动）的子任务之间存在因果依赖——当前动作是否正确取决于前一个子任务是否完成。
- **现有方法的不足**：已有记忆增强VLA要么每帧写入（代价高），要么从演示阶段检索（无法对齐在线进度），要么在子目标事件触发但不执行真正的**子任务到子任务交接**，导致动作专家无法获取紧凑的已完成子任务摘要。
- **整体含义**：WeaveLA提出以**子目标完成事件**为自然时间单位，在冻结的VLA骨干上压缩刚完成的子任务为潜在记忆tokens，并直接注入下一个子任务的动作生成路径，以轻量级接口解决跨子任务依赖问题。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 核心思想
- 利用**子目标完成事件**作为写触发点，在动作侧注入一个紧凑的、每个子任务独立的潜在记忆，从而保留基础策略的短窗口能力，同时增加跨子任务通道。
- 设计三个组件：冻结的感知骨干、查询驱动的Memory Weaver、动作侧记忆条件化。

### 2.2 关键技术细节

#### (1) 子目标事件触发器（Sub-goal-event trigger）
- 仅当子目标完成时（如模拟器中提供边界）才触发记忆写入，写入间隔远低于每帧写入，降低计算开销。

#### (2) 查询驱动Memory Weaver
- 对已完成子任务段提取的视觉、语言、状态特征进行**层归一化**后，通过**8个可学习查询潜在向量**进行单步注意力池化：
  \[
  A = \text{softmax}\left(\frac{QK^\top}{\sqrt{d_h}}\right),\quad m_{k-1}=W_{\text{out}}(AV+Q)
  \]
  输出8个维度为2048的潜在标记（N=8，d_h=1024，输出d=2048）。

#### (3) 动作侧记忆消费
- 以π0.5的Gemma动作专家为例，在原有时间条件AdaRMS之间**插入一个记忆条件AdaRMS**，通过交叉注意力将记忆标记融入动作生成流，使用RoPE和RMSNorm，产生尺度/偏移参数进行调制。
- 冻结预训练骨干，只训练Weaver模块和动作侧投影（约46M参数，占基础模型1.4%）。

#### (4) 分阶段训练
- **Stage 0**：任务集适应，仅动作损失，建立短窗口基础。
- **Stage 1**：引入Weaver与记忆条件化，训练动作损失（K=2子任务窗口）。
- **Stage 2**：加入语义对齐损失（L_align）和对比损失（L_ctr，权重0.05和0.02），可变K∈{2,3,4}。

## 6. 论文的主要结论与发现

- **主要增益定位**：在需要跨子任务信息的任务上（SwingXtimes, StopCube）成功率显著提升，其中**SwingXtimes (N=3) 从0%升至47.8%**（6任务训练）；单执行子任务（N=1）保持约100%，证明机制并非通用容量提升。
- **机制解耦**：触发源可以替换为无外部监督的潜在偏移检测（latent-shift），恢复68%的oracle差距，仅在StopCube上表现差。
- **提取器选择**：单步注意力池化优于Q-Former（后者导致56.3%超时）。
- **瓶颈宽度**：8个标记已饱和，4个不足（因表示崩溃），16个无额外提升。
- **分阶段训练必需**：合并阶段导致潜在表示崩溃（weaver_latent_norm衰减），尽管损失曲线平稳。

## 7. 优点

- **设计自然**：子目标完成事件是重复任务跨子任务依赖的**自然时间单元**，避免了每帧写入的高成本。
- **轻量级**：只增加1.4%参数，冻结骨干，计算开销可忽略。
- **机制对齐证据充分**：分层分析、配对分析、消融实验均一致表明增益仅出现在需要跨子任务通道的任务上，**机制而非偶然**。
- **触发源可解耦**：无需oracle边界，利用对比学习隐式检测边界，提升实用性。
- **分阶段训练分析深入**：识别出潜在表示崩溃的隐形问题，提供监控指标。
- **对比实验全面**：在RoboMME上横向对比14种方法，纵向消融设计维度。

## 8. 不足与局限

- **StopCube上触发源替代失败**：latent-shift在纯符号计数任务上效果差（2% vs oracle 12%），需要进一步学习符号检测器。
- **Q-Former提取器不稳定**：虽未涉及主要方法，但表明注意力池化是合适的选择，Q-Former的失败原因未完全解释。
- **缺乏与感知记忆方法的联合**：如与FrameSamp稠密缓存结合可能进一步提升，论文虽提及但未实验。
- **真实机器人验证有限**：仅单任务20集，未在更多重复任务（如摆动、停止）上验证。
- **边界检测依赖假设**：模拟器实验使用oracle边界，虽latent-shift可替代但依赖于训练好的对比损失，边界检测错误会传播。
- **未讨论与其他输入优化（如动作序列、任务规划）的关系**：聚焦于单一记忆通道，未与高层次规划结合。
